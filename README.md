# Claude-Fable-5

Скилл **`fable5-style`** для Claude — переносит рабочий цикл и стандарты качества Claude Fable 5 на другие модели (например, Claude Opus 4.8). Заточен под стек: Telegram-боты на aiogram 3, Mini Apps, платежи, Google Apps Script и деплой на VPS.

> Скилл влияет на дисциплину, формат и фактическую корректность кода под конкретный стек, но не делает модель умнее — качество рассуждений остаётся на уровне используемой модели.

## Что внутри

**`SKILL.md`** — ядро скилла:

- **Рабочий цикл** (обязательный порядок): понять задачу → открыть справочник → план (3–6 строк) → код целыми файлами/функциями → самопроверка по чеклисту → выдача этапами с «как проверить».
- **Стандарты кода**: секреты только из окружения; `tg_id` → `BigInteger`; деньги → `int` (копейки/звёзды), не `float`; время → UTC-aware; SQL только с параметрами; каждый внешний вызов обрабатывает ошибку.
- **Чеклист самопроверки**: мысленный запуск на конкретном входе, «ничего не выдумано» (каждый метод библиотеки существует), край-кейсы (пустой список, `None`, двойное нажатие, текст > 4096, пользователь заблокировал бота), безопасность, согласованность с проектом, явные границы сделанного.
- **Протокол отладки**: traceback снизу вверх → 1–2 гипотезы → точечное исправление → одной строкой «почему это причина, а не симптом». Shotgun debugging запрещён.
- **Анти-паттерны**: заглушки и псевдокод под видом рабочего кода, выдуманные методы, «должно работать» без способа проверить, абстракции «на вырост».

**`references/`** — справочники с проверенными паттернами. Модель обязана свериться с нужным файлом до написания кода, вместо того чтобы писать по памяти:

| Файл | Что покрывает |
|---|---|
| `aiogram3.md` | aiogram 3: Router, FSM, CallbackData-фабрики, middleware, рассылки, типовые ошибки миграции с v2 + SQLAlchemy 2.0 async, Alembic |
| `telegram-miniapp.md` | Mini App / WebApp: валидация initData на FastAPI, минимальный фронт, статика, ключевые факты |
| `payments.md` | Платежи: Stars (XTR) vs классические провайдеры, invoice → pre_checkout → successful_payment, возвраты, идемпотентность по `charge_id` |
| `scheduler.md` | APScheduler: cron/interval/date-задачи, рассылки по расписанию, персистентные напоминания через БД, таймзоны |
| `testing.md` | pytest + pytest-asyncio: тонкие хендлеры и логика в `services/`, in-memory SQLite на каждый тест, моки `Message` |
| `apps-script.md` | Google Apps Script: батчи, отчёты в Telegram, устойчивость к изменению таблиц, лимиты |
| `vps-deploy.md` | Деплой: VPS, systemd + venv + .env, PostgreSQL, nginx, polling vs webhook, обновление, бэкапы |

## Как это работает

Скилл срабатывает на **любую** задачу разработки — код, ревью, отладку, рефакторинг, вопросы по деплою, даже маленькие сниппеты. Перед написанием кода модель открывает подходящий справочник по таблице маршрутизации в `SKILL.md`; для задач на стыке (например, бот + платежи) — несколько сразу.

## Установка

**Claude Code:**

```bash
mkdir -p ~/.claude/skills/fable5-style
cp SKILL.md ~/.claude/skills/fable5-style/
cp -r references ~/.claude/skills/fable5-style/
```

**Claude.ai / приложение Claude:** упакуйте скилл в zip и отправьте в чат, затем нажмите **Save skill** на карточке файла:

```bash
mkdir fable5-style && cp SKILL.md fable5-style/ && cp -r references fable5-style/
zip -r fable5-style.skill fable5-style
```

## Как дорабатывать

Всё — обычный Markdown:

1. Новый проект или технология → добавьте файл в `references/` с проверенными паттернами и типовыми граблями.
2. Допишите строку в таблицу «Маршрутизация по справочникам» в `SKILL.md` (триггеры → файл).
3. При необходимости обновите `description` в шапке `SKILL.md`, чтобы скилл срабатывал на новые темы.
4. Для установки в Claude.ai пересоберите `.skill` (см. раздел «Установка»).

Принцип справочников: короткие рабочие примеры кода + раздел «Грабли» с типовыми ошибками — модель берёт готовый корректный паттерн вместо воспоминаний.
