# Эталонные задачи — проверка, что скилл реально работает

Как прогонять: в Claude Desktop открыть **новый чат** с включённым скиллом и дать задачу как есть. Для сравнения — повторить в чате с выключенным скиллом. Скилл работает, если проходят **5 из 6** проверок (у моделей есть разброс между запусками).

Файл не входит в `.skill`-архив — он для человека, а не для модели.

## 1. Утёкший токен

> Я случайно закоммитил токен бота в git, уже удалил файл следующим коммитом. Всё нормально?

**Ожидание** (`git.md`): первым пунктом — токен скомпрометирован, отозвать через BotFather (Revoke), потому что история git хранит всё; удаление файла следующим коммитом проблему не решает.
**Провал**: «да, раз удалили — всё в порядке» или совет просто добавить файл в `.gitignore`.

## 2. Платёж

> Напиши команду /buy для продажи месячной подписки на премиум в моём боте.

**Ожидание** (`payments.md`): Stars (XTR) для цифрового товара; хендлер `pre_checkout_query`; сохранение `telegram_payment_charge_id`; защита от повторной выдачи (уникальность charge_id); цена в `int`.
**Провал**: `provider_token` для цифрового товара без обсуждения, нет `pre_checkout_query`, цена во `float`, нет идемпотентности.

## 3. Диагноз, а не правки

> Бот на сервере молчит, локально всё работает. В чём может быть дело?

**Ожидание** (`debugging.md`, `agentic.md`): это запрос диагноза — модель даёт 1–2 гипотезы и конкретные проверки (`journalctl -u mybot -n 50`, `getWebhookInfo`, жив ли сервис), а не переписывает код и не выдаёт «исправленную версию».
**Провал**: сразу код «попробуй так», без вопросов о логах и без гипотез.

## 4. Расписание

> Сделай, чтобы бот каждый день в 9 утра слал всем пользователям напоминание.

**Ожидание** (`scheduler.md`, `aiogram3.md`): APScheduler с **явной таймзоной**; рассылка с паузами и обработкой `TelegramForbiddenError`; запуск планировщика внутри running event loop.
**Провал**: `while True: sleep(86400)`, рассылка без пауз и обработки блокировок, наивное время без таймзоны.

## 5. Ошибка из трейсбека

> Лови ошибку: sqlalchemy.exc.MissingGreenlet: greenlet_spawn has not been called; can't call await_only() here. Падает когда открываю профиль игрока.

**Ожидание** (`debugging.md`): диагноз — обращение к lazy-атрибуту связи вне async-контекста сессии; просьба показать модель/хендлер или гипотеза про `relationship` без eager load; точечный фикс (`selectinload` или явный запрос).
**Провал**: совет переписать всё на sync, «обнови библиотеку», правка наугад без объяснения причины.

## 6. Формат любого ответа (проверяется на задачах 1–5)

**Ожидание** (`communication.md`, `verification.md`): ответ начинается с итога, а не с «Давайте разберёмся»; к коду приложено «как проверить» конкретной командой/действием; названы границы («миграцию не писал», «код не запускал»); нет заглушек `# тут логика`.
**Провал**: лекция до сути, «должно работать» без способа проверить, псевдокод.

## Проверка версии установленного скилла

Спросить в чате: «какая версия скилла fable5-style?» — модель прочитает строку «Версия: X.Y (дата)» из SKILL.md. Если версия старее, чем в репозитории, — переустановить скилл (README → Установка).
