# Хуки Claude Code — принудительная дисциплина

Скилл — это просьба к модели; хук — это гарантия от среды. Этот набор переводит
два главных правила fable5-style из просьб в механику:

| Хук | Событие | Что принуждает |
|---|---|---|
| `post-edit.sh` | после каждой правки `.py`-файла | файл компилируется и проходит `ruff` — иначе модель обязана исправить до продолжения |
| `stop-verify.sh` | перед завершением хода | `pytest` зелёный — иначе ход не завершается, модель чинит или честно докладывает |

## Установка в проект бота

Из корня вашего проекта (Windows — те же команды в Git Bash, либо скопируйте файлы вручную):

```bash
mkdir -p .claude/hooks
cp <путь-к-этому-репо>/claude-code-setup/hooks/*.sh .claude/hooks/
chmod +x .claude/hooks/*.sh
```

Содержимое `settings-fragment.json` добавьте в `.claude/settings.json` проекта
(если файла нет — просто скопируйте фрагмент целиком как `settings.json`).

Шаблон `CLAUDE.md.template` скопируйте в корень проекта как `CLAUDE.md` и
заполните: это факты о вашем проекте (команды, правила, запреты), которые
скилл знать не может.

## Проверка, что хуки работают

1. Попросите Claude Code внести в любой `.py`-файл заведомую синтаксическую
   ошибку — после правки модель должна получить отказ хука и исправить файл.
2. Сломайте один тест и попросите мелкую правку — при завершении хода модель
   должна отчитаться о красном тесте, а не промолчать.

## Замечания

- `stop-verify.sh` защищён от бесконечного цикла (`stop_hook_active`) и
  срабатывает только если в проекте есть каталог `tests/` и установлен pytest.
- Хуки локальны для проекта (`.claude/` в репозитории проекта) — на другие
  проекты не влияют.
