# Claude-Flame-5

Скилл **`fable5-style`** для Claude — переносит рабочий цикл и стандарты качества Claude Fable 5 на другие модели (например, Claude Opus 4.8).

## Что внутри

- **Рабочий цикл**: понять задачу → свериться со справочником → план → код → обязательная самопроверка → выдача этапами.
- **Чеклист самопроверки**: запустится как есть, ошибки обработаны, безопасность (токены, SQL, initData), согласованность с проектом, ничего не выдумано.
- **Протокол отладки**: гипотезы вместо переписывания всего подряд.
- **Справочники** (`references/`) с проверенными паттернами — модель берёт готовый корректный код вместо воспоминаний:
  - `aiogram3.md` — aiogram 3 (Router, FSM, CallbackData-фабрики, типовые ошибки миграции с v2) + SQLAlchemy 2.0 async
  - `telegram-miniapp.md` — валидация initData на FastAPI, минимальный фронт, ключевые факты
  - `apps-script.md` — батчи, отчёты в Telegram, устойчивость к изменению таблиц, лимиты
  - `vps-deploy.md` — systemd + venv + .env + PostgreSQL, обновление, polling vs webhook

> Скилл влияет на дисциплину, формат и фактическую корректность кода под конкретный стек, но не делает модель умнее — качество рассуждений остаётся на уровне используемой модели.

## Установка

**Claude.ai / приложение Claude:** отправьте файл `fable5-style.skill` в чат и нажмите **Save skill** на карточке файла.

**Claude Code:**

```bash
cp -r fable5-style ~/.claude/skills/
```

## Структура

```
Claude-Flame-5/

└── fable5-style/
    ├── SKILL.md              # рабочий цикл, чеклисты, формат
    └── references/           # справочники по стеку
        ├── aiogram3.md
        ├── telegram-miniapp.md
        ├── apps-script.md
        └── vps-deploy.md
```

## Как дорабатывать

Всё — обычный Markdown. Новый проект со своими конвенциями → добавьте файл в `references/` и строку в таблицу «Справочники» в `SKILL.md`. После правок пересоберите `.skill` (это zip папки `fable5-style/`).
