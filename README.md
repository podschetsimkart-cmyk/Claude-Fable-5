# Claude-Fable-5

Скилл **`fable5-style`** для Claude — переносит полную рабочую дисциплину Claude Fable 5 на другие модели (например, Claude Opus 4.8): доведение задачи до проверенного результата, обязательная проверка запуском, честный отчёт, коммуникация «итог первой строкой» — плюс справочники с проверенными паттернами под стек: Telegram-боты на aiogram 3, Mini Apps, платежи, Google Apps Script, деплой на VPS.

> Честное ограничение: скилл переносит дисциплину, формат и фактическую корректность под конкретный стек, но не делает модель умнее — качество рассуждений остаётся на уровне используемой модели.

## Из чего состоит

**`SKILL.md`** — ядро: пять главных принципов (итог первой строкой; «проверено» = запущено; автономность; довести ход до конца; честность), обязательный рабочий цикл из 8 шагов, стандарты кода, чеклист самопроверки из 7 пунктов, краткие протоколы отладки и формата ответов, список запрещённых анти-паттернов и маршрутизация по справочникам.

**`references/` — поведение** (как Fable 5 ведёт задачу):

| Файл | Что покрывает |
|---|---|
| `agentic.md` | Автономность: когда делать без вопроса, а когда спросить; запрет заканчивать ход планом/обещанием; ошибки не завершают ход; чтение перед правкой, улики перед действием; параллельные инструменты |
| `verification.md` | Иерархия доказательств от end-to-end прогона до мысленного запуска; что значит «прогнать сценарий»; честный отчёт (упало — показать, пропустил — сказать); definition of done |
| `communication.md` | Итог первой строкой; читаемость важнее краткости; форма под вопрос; калибровка под читателя; обязательные элементы отчёта |
| `code-quality.md` | Согласованность с проектом; дисциплина комментариев; простота против абстракций «на вырост»; надёжность внешних вызовов; безопасность |
| `debugging.md` | Порядок от симптома к причине: воспроизвести → гипотезы → наблюдение → точечная правка → повторить сценарий; ловушка узнавания; запрет shotgun debugging; типовые причины по стеку |
| `git.md` | Гигиена коммитов и `.gitignore` до первого коммита; атомарные коммиты и внятные сообщения; ветки и откаты (`revert` vs `reset`); утёкшие токены; деплой через `git pull` без правок на сервере |

**`references/` — стек** (проверенные паттерны, модель сверяется до написания кода):

| Файл | Что покрывает |
|---|---|
| `aiogram3.md` | aiogram 3: Router, FSM, CallbackData-фабрики, middleware, рассылки, типовые ошибки миграции с v2 + SQLAlchemy 2.0 async, Alembic |
| `telegram-miniapp.md` | Mini App / WebApp: валидация initData на FastAPI, минимальный фронт, статика, ключевые факты |
| `payments.md` | Платежи: Stars (XTR) vs классические провайдеры, invoice → pre_checkout → successful_payment, возвраты, идемпотентность по `charge_id` |
| `scheduler.md` | APScheduler: cron/interval/date-задачи, рассылки по расписанию, персистентные напоминания через БД, таймзоны |
| `testing.md` | pytest + pytest-asyncio: тонкие хендлеры и логика в `services/`, in-memory SQLite на каждый тест, моки `Message` |
| `apps-script.md` | Google Apps Script: батчи, отчёты в Telegram, устойчивость к изменению таблиц, лимиты |
| `vps-deploy.md` | Деплой: VPS, systemd + venv + .env, PostgreSQL, nginx, polling vs webhook, обновление, бэкапы |
| `logging.md` | Логи и мониторинг: что логировать (и что нельзя), алерты админу с кулдауном, journalctl, healthcheck |
| `cicd.md` | CI/CD: GitHub Actions — тесты и ruff на каждый пуш, автодеплой на VPS по SSH после зелёных тестов |

## Как это работает

Скилл срабатывает на **любую** задачу разработки — код, ревью, отладку, рефакторинг, вопросы по деплою, даже маленькие сниппеты. Поведенческие принципы действуют всегда; детальные файлы модель открывает по фазе работы (ведение задачи → `agentic.md`, прислали ошибку → `debugging.md`, сдача результата → `verification.md` и `communication.md`), а стековые — по таблице маршрутизации перед написанием кода. Для задач на стыке (бот + платежи) читается несколько справочников сразу.

## Установка

**Самый простой способ** — скачать готовый архив [`fable5-style.skill` из Releases](https://github.com/podschetsimkart-cmyk/Claude-Fable-5/releases/latest): он автоматически пересобирается при каждом изменении на `main`. Дальше — отправить его в любой чат Claude и нажать **Save skill** (старую версию скилла перед этим удалить в настройках). Проверить установленную версию: спросить в чате «какая версия скилла fable5-style?».

Ручные способы:

### Claude Code

**Windows (PowerShell):**

```powershell
New-Item -ItemType Directory -Force "$HOME\.claude\skills\fable5-style" | Out-Null
Copy-Item SKILL.md "$HOME\.claude\skills\fable5-style\"
Copy-Item -Recurse -Force references "$HOME\.claude\skills\fable5-style\references"
```

**Linux / macOS:**

```bash
mkdir -p ~/.claude/skills/fable5-style
cp SKILL.md ~/.claude/skills/fable5-style/
cp -r references ~/.claude/skills/fable5-style/
```

### Claude.ai / приложение Claude (Desktop)

Упакуйте скилл в архив и отправьте его в любой чат Claude, затем нажмите **Save skill** на карточке файла. Если старая версия уже установлена — сначала удалите её в настройках, чтобы не было дубля.

**Windows (PowerShell):**

```powershell
New-Item -ItemType Directory -Force fable5-style | Out-Null
Copy-Item SKILL.md fable5-style\
Copy-Item -Recurse -Force references fable5-style\references
Compress-Archive -Path fable5-style -DestinationPath fable5-style.zip -Force
Rename-Item fable5-style.zip fable5-style.skill -Force
```

**Linux / macOS:**

```bash
mkdir fable5-style && cp SKILL.md fable5-style/ && cp -r references fable5-style/
zip -r fable5-style.skill fable5-style
```

## Проверка, что скилл работает

В `benchmark.md` — 6 эталонных задач с ожидаемым поведением (утёкший токен → revoke; «бот молчит» → диагноз, а не переписывание; и т.д.). Прогоните их в Claude Desktop с включённым скиллом — и, для сравнения, без него.

## Как дорабатывать

Всё — обычный Markdown:

1. Новый проект или технология → добавьте файл в `references/` с проверенными паттернами и типовыми граблями.
2. Допишите строку в нужную таблицу «Маршрутизация по справочникам» в `SKILL.md` (триггеры → файл).
3. При необходимости обновите `description` в шапке `SKILL.md`, чтобы скилл срабатывал на новые темы.
4. Для установки в Claude.ai пересоберите `.skill` (см. раздел «Установка»).

Принцип справочников: короткие рабочие примеры + раздел «Грабли»/«Анти-паттерны» с типовыми ошибками — модель берёт готовый корректный паттерн вместо воспоминаний.

После правок поднимите версию в строке «Версия: X.Y (дата)» в `SKILL.md` и допишите строку в таблицу версий ниже — CI проверит консистентность и пересоберёт архив в Releases автоматически.
